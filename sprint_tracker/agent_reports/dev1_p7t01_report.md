## Architecture Decisions

### VPC
- CIDR `10.0.0.0/16`, 2 public subnets (10.0.1/2.0/24), 2 private subnets (10.0.10/11.0/24)
- NAT gateway per AZ for private subnet egress HA
- 5 security groups with strict least-privilege ingress: alb-sg, api-sg, web-sg, rds-sg, redis-sg

### ECS / ALB
- Fargate cluster with Container Insights enabled
- Task definitions use `secrets` blocks (Secrets Manager) — no plaintext env vars for sensitive data
- ALB: HTTP→HTTPS redirect, TLS 1.3 policy, `/api/*` listener rule routes to API target group, everything else goes to web
- `deployment_circuit_breaker` with rollback enabled on both services
- `lifecycle { ignore_changes = [task_definition, desired_count] }` — CI manages deploys after initial apply
- ALB access logs stored in S3 (90-day expiry)

### RDS
- `db.t3.medium`, PostgreSQL 16, 20 GB gp3 with autoscale to 100 GB
- `multi_az = true`, 7-day backup retention
- Master password generated by `random_password` (32 chars), stored as `DATABASE_URL` in Secrets Manager after instance creation
- Performance Insights enabled, slow query logging (>1s), `pg_stat_statements` preloaded
- `deletion_protection = true`

### ElastiCache
- Redis 7, `cache.t3.micro`, single-node (upgrade to `aws_elasticache_replication_group` at Section 9 scale)
- `maxmemory-policy = allkeys-lru` (suits leaderboard sorted sets + session caching)
- Snapshot retention 3 days

### S3 + CloudFront
- Private S3 bucket (`koblio-assets-{env}`), versioning + AES256 encryption
- CloudFront OAC (not deprecated OAI) — sigv4 signing
- `PriceClass_200` — US/EU/Asia; excludes AU/South America to reduce cost
- `compress = true`, HTTPS-only viewer protocol
- CORS configured for `*.koblio.com` browser uploads

### IAM
- Two separate roles: `ecsTaskExecutionRole` (ECS agent — ECR pull + CloudWatch + Secrets) and `ecsTaskRole` (app runtime — Secrets read + S3 assets + CloudWatch metrics)
- Custom `koblio-{env}-ecs-secrets-read` policy scoped to exact secret ARNs

### Secrets Manager
- 6 secrets: `koblio/{env}/db-password`, `auth0`, `stripe`, `sendgrid`, `sentry`, `redis`
- No plaintext values committed — populate manually after apply per instructions in secrets.tf comments
- All secret ARNs exported as outputs

### Backend
- S3 state bucket with versioning + AES256 encryption + public access blocked
- DynamoDB lock table with PAY_PER_REQUEST billing
- Both have `lifecycle { prevent_destroy = true }`
- Bootstrap instructions documented in backend.tf
